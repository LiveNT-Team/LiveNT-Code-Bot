## Система приветствий
Функционал автоматического приветствия новых участников сервера. При входе пользователя бот генерирует персонализированное приветственное сообщение с использованием ИИ и публикует его в специальном канале для приветствий. Сообщение включает упоминание (@mention) нового участника. Система поддерживает возможность активации и деактивации через настройки бота. Настроить канал для приветствий можно так же через настройки бота.
## Система настроек
Система настроек поведения бота. Эта система включает в себя активацию и деактивацию отдельных систем бота, изменение значений параметров (например, изменение канал для приветствий), и удобный вывод текущих настроек бота.
## Система чата с ИИ ассистентом
Система чата с ИИ ассистентом с возможностью выбрать одну из заготовленных инструкций. Заготовленная инструкция (далее, "личность ИИ") - объект, описывающий поведение ии, температуру, максимальное количество токенов ответа. В боте присутствует несколько заготовленных личностей ИИ, от ассистента до разработчика. Система реагирует на два события: сообщения содержащие промпт, отправляемые в определённый канал, или сообщения, содержащие упоминание бота и промпт. После этого система генерирует ответ и отправляет его пользователю который запросил его. Система поддерживает возможность активации и деактивации через настройки бота. Настроить канал для чата с ИИ ассистента можно так же через настройки бота.
## Система изменения личности ИИ
Бот предоставляет функционал для изменение личности ИИ из списка доступных. Идентификатор личности ИИ привязывается к пользователю, и все ответы от ИИ должны быть сгенерированы с учётом выбранной личности бота. Система зависит от системы чата с ИИ ассистентом, деактивация системы чата с ИИ приведёт к деактивации обоих систем.
## Система счётчика отправленных сообщений
Каждое отправленное сообщение от пользователя увеличивает счетчик сообщений у этого пользователя. Бот предоставляет функционал для просмотра счётчика сообщений.
## Система автоматической выдачи ролей
Система автоматически выдаёт установленную роль (далее именуемую как роль активиста) участнику, набравшему установленное число сообщений. Система поддерживает возможность активации и деактивации через настройки бота.
